Famazone-Store

Famazone-Store is one of my first projects built with Angular, where I focused on learning components, services, and routing. The main goal was to create a basic e-commerce application with a shopping cart and navigation.

The project is built with Angular, using TypeScript, Bootstrap, and Angular Routing. The main structure consists of dynamic components such as app-navbar, app-footer, products, cart, product-details, and others, which are organized in separate folders. This ensures modularity and scalability of the application. For managing the shopping cart, I implemented a Cart Service, which allows adding, removing products, and maintaining the cart's state.

Navigation between pages is handled through Angular Router, utilizing <router-outlet> to load only the necessary components without reloading the entire page.

Since the primary focus of this project was on leveraging Angular's features, I deliberately did not spend much time on styling. Instead, I concentrated on the code structure, dynamic content rendering, and the proper use of services and routing.

Like all my learning projects, this was an important step in mastering the Angular framework and understanding the client-server architecture of web applications.

Meal Planner

Meal Planner is my most advanced and large-scale project, where I worked with Symfony. It is fully functional and follows the MVC (Model-View-Controller) architecture, giving me a deep understanding of modern web application development.

In this project, I:

  • Learned how to work with Symfony forms and customize forms to adapt them to specific needs.
  • Implemented AJAX with `XMLHttpRequest`: for example, during registration, the system instantly checks if the entered email already exists in the database and notifies the user. This greatly improves the UI/UX.
  • Developed a photo upload feature in forms, allowing users to preview their selected image and its size before submission.
  • Mastered database operations using Doctrine, repositories, and entities, enabling efficient data management, table relationships, and executing complex queries without writing raw SQL.
  • Gained extensive experience with Git, learning how to create branches, save different versions of code, and push them to a remote repository. Git became an essential tool for both team collaboration and my individual improvements after the project's completion.

How the project works

  • Users can:
    • Register, log in, create, and edit recipes.
    • Plan their meals and review saved meal plans.
  • Technologies used:
    • Symfony (controllers, services, forms, repositories, Doctrine ORM).
    • JavaScript + AJAX (`XMLHttpRequest`) for instant form validation and dynamic data updates.
    • Twig for rendering pages and interacting with the backend.
  • What is needed for working with Symfony:
    • PHP, Composer, Doctrine ORM for database management.
    • Twig for templating.
    • Symfony console commands for generating controllers, migrations, and handling database operations.

Anyone reviewing this project can log in as a regular user by registering and signing in, then creating a new recipe. Additionally, they can access the project as an admin:

  • Email: admin@mail.com
  • Password: 123123

As an administrator, they can navigate to the Pending Recipes section, where they can approve, reject, or edit submitted recipes, managing content moderation efficiently.

This project was developed as part of a team collaboration, and after completion, I significantly refined and improved it independently, as reflected in my commit history on GitHub. This was a major step in my development as a Full-Stack Developer, and I continue enhancing this application by adding new features.

Traffic Analysis Dashboard

Traffic Analysis Dashboard is an interactive Angular application designed to visualize road accident data using a REST API. The main goal is to provide users with powerful filtering and analytical tools for working with large data sets.

The project features dynamic data loading, filtering by criteria (year, gender, location, etc.), and custom 3D chart rendering using the Chart.js library extended with custom plugins. Chart integration is handled via the ng2-charts wrapper, allowing full control over chart behavior and rendering.

All data is fetched in real-time from the backend via API requests, ensuring the dashboard presents up-to-date and realistic statistics for analytical insights. This project showcases a fully functional client-side analytics system with a focus on interactivity, data accuracy, and user experience.
